Contrast Variations

Madek comes with an extra, high-contrast variation accessible on-demand via user menu.

Default vs. Contrasted Variation.

There are 2 application stylesheets in the /app/assets/stylesheets/ folder. The first one, application.sass, is the primary one which is being loaded by default. The other, application-contrasted.sass, replaces the primary one only if user has toggled high contrast mode via the user menu.

application.sass uses _colors.sass to define default UI color palette whereas appliaction-contrasted.sass makes use of _colors-contrasted.sass where high contrast color variables are defined. When expanding color schemes in the future (for example to add another color to replace default green Madek instance), make sure to add new color variables to both color files located in /app/assets/stylesheets/ folder.

Color Variations

Madek comes with 4 default color schemes:

  • Green
  • Blue
  • Red
  • Monochrome

Swithing between color schemes is as simple as altering scheme variables in the _config stylesheet located in the /app/assets/stylesheets folder: 'true' to any other $scheme-% variable and disable the currently set one by changing its value to false.

Color schemes are defined in both _colors-all and _colors-projection placed in the /app/assets/stylesheets/ folder. The first one defines the default color palette whereas the latter defines the contrasted variation accessible via the user menu (documented above).
